2. CORE AUDIT QUESTIONS & ANSWERS
---------------------------------

### Q1: Does the DNN actually outperform XGBoost?
*   **ANSWER: NO.** 
    *   The MultiTaskDNN achieves a crop prediction test accuracy of **97.58%**.
    *   XGBoost and Random Forest achieve **99.39%** accuracy.
    *   SVM achieves a stellar **99.70%** accuracy.
    *   The DNN lags behind standard tree architectures and SVM by **1.81% to 2.12%** while requiring 10x the training duration.

### Q2: Is Deep Learning justified for this dataset size/type?
*   **ANSWER: BRUTALLY NO.**
    *   The dataset is classic tabular soil telemetry (a few thousand rows). Neural networks struggle to establish clean axis-aligned split boundaries compared to decision trees on small tabular manifolds.
    *   Furthermore, utilizing TensorFlow/Keras forces the serving container to load a heavy runtime, adding **over 1.5GB of RAM overhead** and introducing potential thread blocking. 
    *   The use of a DNN here is "AI Theater" — pure decoration to check a buzzword box.

### Q3: Which model is most stable under noise?
*   **ANSWER: Support Vector Machine (SVM) and RandomForest.**
    *   Random Forest averaging minimizes variance under localized soil anomalies.
    *   SVM with an RBF kernel scales excellently because it maximizes the geometric margin between decision boundaries, protecting against manual input errors.
    *   XGBoost is highly sensitive to noise in early iterations unless aggressively regularized (`reg_lambda` set high).

### Q4: Which model generalizes best to unseen data?
*   **ANSWER: SVM and RandomForest.**
    *   SVM achieves **99.70%** accuracy on the test set, indicating zero generalization loss from its 98.18% validation score.
    *   Gradient Boosting shows a slight validation-to-test drop (99.70% -> 98.79%).

### Q5: Which model calibrates prediction confidence best?
*   **ANSWER: Support Vector Machine (with Platt Scaling) and XGBoost.**
    *   XGBoost `predict_proba` returns highly calibrated outputs based on log-loss optimization, which fits neatly into our weather and knowledge-graph hybrid scoring equations.
    *   Logistic Regression produces smooth, but occasionally under-confident, probabilities.

### Q6: Is the Stacked Ensemble complexity justified?
*   **ANSWER: DEFINITELY NO.**
    *   The Stacked Ensemble (RF + XGBoost -> Logistic Regression meta-learner) scores **99.70%** test accuracy.
    *   This is the *exact same* score achieved by the simple Support Vector Machine (SVM) alone.
    *   The ensemble requires saving, loading, and running predictions on three separate estimators in sequence, taking **7.70 seconds** to train versus SVM's **0.11 seconds**. 
    *   This is textbook architectural inflation.

4. SECTION CONCLUSION
---------------------
*   **ACTION:** Decommission the MultiTaskDNN and the Stacked Ensemble immediately.
*   **ACTION:** Restructure the inference engine to utilize **SVM** for crop predictions and **XGBoost** for soil fertility predictions. This cuts container footprint by 90% and latency by over 80%.
