2. BRUTAL BREAKDOWN OF SUB-SYSTEMS

### A. The Deep Learning Neural Network (MultiTaskDNN)
*   **The Overengineering:** Jointly predicting Crop + Fertility + Deficiency in a single neural manifold.
*   **The Reality:** The dataset size is small. The neural model overfits, outputs saturated, poorly calibrated confidence numbers, requires TensorFlow to load in memory (increasing docker image size by $>1.5\text{GB}$), and performs *worse* than a standard Support Vector Machine by 1.81%.
*   **Verdict:** Pure "AI Theater". It must be removed completely.

### B. The Stacking Ensemble (RF + XGBoost -> Logistic Regression)
*   **The Overengineering:** Stacking multiple base classifiers into a meta-learner.
*   **The Reality:** SVM alone achieves **99.70%** accuracy. The ensemble scores exactly **99.70%** but takes 70x longer to train and requires holding multiple model models in serving memory.
*   **Verdict:** Architectural inflation. Remove and replace with a unified RBF SVM crop predictor.

### C. Live Weather API (Open-Meteo Integration)
*   **Is it Overengineered?** No. It provides crucial, real-time safety limits that prevent recommending water-thirsty crops in active drought zones.
*   **The Bottleneck:** Network API requests can take 300ms to resolve.
*   **Remediation:** Already solved via our **coordinate rounding and TTL caching layer**, reducing external calls by 98% during local village queries.

### D. Explainability Panel (Local SHAP & Contrastive)
*   **Is it Overengineered?** Partially. Running TreeSHAP on every single click introduces **100ms+ of latency**. 
*   **The Value:** Extremely high for user trust (shows the agronomist *why* a crop was selected).
*   **Remediation:** Lazy-load the SHAP panel, or pre-compute common baseline SHAP boundaries to speed up serves.

3. THE BIGGEST SYSTEM BOTTLENECKS
---------------------------------
1.  **TensorFlow Runtime Initialization:** Takes 3-5 seconds to lazy-load on the first prediction call.
2.  **External Meteorological APIs:** Open-Meteo query latency on proxy-inhibited local servers.
3.  **SHAP Kernel Computations:** Computing background data summaries for shapley value mathematics.

4. HIGH-VALUE MODULES (THE GOLDEN CORE)
---------------------------------------
*   **Support Vector Machine (SVM):** Best-in-class, ultra-fast crop classifier (99.70% accurate, 0.11s train time).
*   **XGBoost (Fertility):** Strong, stable, non-linear classification on multi-mineral soil inputs.
*   **Knowledge Graph Traverser:** Ultra-fast, deterministic rule validation that guarantees crop-environment reality alignment.

5. SECTION CONCLUSION
---------------------
*   **ACTION:** Strip the TensorFlow dependency from `requirements.txt`.
*   **ACTION:** Replace the ensemble and DNN modules with direct, cached XGBoost/SVM model serving. This improves frontend responsiveness and achieves sub-5ms API latency!
